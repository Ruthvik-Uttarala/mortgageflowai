# Interview Talking Points

- MortgageFlowAI is not presented as affiliated with AngelAi, Celligence, or Sun West. It is an interview project inspired by public product themes.
- The app focuses on real mortgage work: status, missing documents, conditions, verification recovery, realtor readiness, loan officer scenarios, and auditability.
- The strongest product choice is explainability. Every recommendation shows the decision, why, data used, missing data, next action, escalation path, and fairness rule.
- The engineering design maps directly to the job description: React/Next.js, Node.js API, AWS Lambda shape, Python AI cells, C++ systems logic, Java JNI, MongoDB, Oracle SQL, Vercel, and GitHub.
- The project avoids real PII and uses synthetic seed data only.
- The rate lock parser is deliberately a small internal utility, not the product hero, because mortgage workflows often include rate and lock period scenarios and the assessment tests string parsing.
- The project is built to show product taste and systems judgment together: clean UI, realistic workflows, deterministic decision cells, and deployment-ready boundaries.
